## [핵심 솔루션] B2B 그로스 루프 설계 전략

### ✅ **결론: B2B에서도 바이럴 루프는 충분히 가능합니다**

단, B2C와 달리 **"팀 단위 네트워크 효과"**와 **"조직 내 권장 메커니즘"**을 설계해야 합니다.

---

## 🔄 [3단계 그로스 루프 설계]

### **STAGE 1: 제품 내 자연스러운 확산 장치 (Product-Viral Loop)**

#### 1️⃣ **협업 초대 필수화 구조**
```
[현재 추정 시나리오]
법무팀 A → 계약서 검토 완료 → (끝)

[개선안]
법무팀 A → 계약서 검토 요청 → 인사팀 B에게 승인 요청 전송
→ 인사팀 B가 봇 없으면 "이 문서를 보려면 Collab-Bot 설치" 안내
→ 원클릭 설치 유도
```

**적용 방법:**
- 문서 공유 시 **비사용자에게 "미리보기 + 설치 CTA"** 자동 표시
- "이 문서는 Collab-Bot으로 작성되었습니다" 워터마크

#### 2️⃣ **팀 간 가시성 확보 (Internal Marketplace)**
```
예시: Slack App Directory 방식 내재화
"우리 회사에서 이미 법무팀, 재무팀이 사용 중입니다" 표시
→ 인사팀의 심리적 진입장벽 ↓
```

**기술 구현:**
- 같은 도메인(@회사명.com) 사용자 감지
- "귀사의 3개 부서가 이미 사용 중입니다" 대시보드 노출

---

### **STAGE 2: 인센티브 기반 추천 루프 (Referral Loop)**

#### 🎁 **B2B에 맞는 리워드 설계**

| 유형 | B2C 방식 (❌) | B2B 맞춤 방식 (✅) |
|------|---------------|-------------------|
| 개인 혜택 | 추천 시 1만원 할인 | 팀 추천 시 **팀 전체 프리미엄 기능 1개월 무료** |
| 즉시성 | 친구 가입 즉시 포인트 | 추천팀 **유료 전환 시 추천팀도 혜택** |
| 구조 | 1:1 추천 | **1:N 팀 단위 추천** |

#### 📌 **추천 메커니즘 예시**
```
"다른 팀에 추천하기" 버튼
→ 이메일/메신저로 초대장 발송
→ 초대받은 팀이 3개 문서 이상 처리 시
→ 추천한 팀에게 "고급 템플릿 3종 무료 제공"
```

**핵심:** 금전적 인센티브보다 **"업무 효율성을 높이는 실용적 혜택"**이 B2B에서 효과적

---

### **STAGE 3: 콘텐츠 기반 자연 유입 루프 (Content Loop)**

#### 📄 **템플릿 마켓플레이스 구축**
```
사용자가 만든 계약서 템플릿 공유 기능
→ 다른 기업이 검색엔진에서 "근로계약서 자동화" 검색
→ Collab-Bot 템플릿 페이지 노출
→ "이 템플릿 사용하려면 가입 필요" 전환
```

**사례 참고:**
- **Notion**: 템플릿 갤러리로 월 100만 신규 유입
- **Figma**: 커뮤니티 디자인 파일 공유로 바이럴 확산

#### 🏆 **임파워먼트 루프 (사용자 성과 공유)**
```
"Collab-Bot 도입 후 계약서 검토 시간 70% 단축" 
→ 고객 성공 사례 자동 생성 및 LinkedIn 공유 제안
→ 해당 고객사 임원진/팀장급이 자발적으로 공유
→ 같은 업계 타사 관심 증가
```

## 🛠️ [Step-by-Step 실행 가이드]

### **Phase 1: 즉시 실행 (1~2주 내)**

#### ✅ **최소 기능으로 바이럴 테스트**
```
1. 문서 공유 시 "Powered by Collab-Bot" 링크 삽입 (개발 2일)
2. 같은 회사 내 사용팀 수 표시 기능 (개발 3일)
3. "다른 팀 초대" 버튼 + 이메일 초대장 (개발 3일)
```

#### 📊 **측정 지표 설정**
- **K-Factor** = (초대 발송 수 × 전환율) ÷ 초대한 사용자 수
- 목표: K-Factor **0.3 이상** (1명이 0.3명을 데려오면 → 시간이 지나면 지수적 성장)

---

### **Phase 2: 검증 및 최적화 (1~2개월)**

#### 🔬 **A/B 테스트 항목**
| 변수 | A안 | B안 |
|------|-----|-----|
| 초대 메시지 | "같이 써보자" | "이 문서 보려면 설치 필요" |
| 리워드 | 프리미엄 기능 무료 | 템플릿 3종 제공 |
| 초대 타이밍 | 가입 직후 | 5번째 문서 작성 후 |

#### 📈 **핵심 개선 포인트**
```
현재 팀 간 전파 속도 느림 → 원인 분석 필요
가설 1: 팀장급 의사결정권자가 모름 → 자동 리포트 기능 추가
가설 2: 타팀과 협업 접점 없음 → 크로스팀 워크플로우 템플릿 제공
```

---

### **Phase 3: 스케일업 (3~6개월)**

#### 🚀 **Enterprise Growth Loop**
```
1단계: 한 팀(10명) 무료 사용
2단계: 타팀 3곳 추가 도입 시 → 회사 전체 Enterprise 플랜 제안
3단계: 전사 도입 → "우수 사례" 콘텐츠 제작 → 외부 확산
```

#### 💼 **세일즈 연계 전략**
```
마케팅팀 역할: 개별 팀 무료 트라이얼 유도
세일즈팀 역할: 3팀 이상 사용 기업에 Enterprise 제안
→ Bottom-Up(현장) + Top-Down(경영진) 동시 공략
```

## ⚠️ [리스크 체크리스트]

### 🚨 **B2B 바이럴의 함정**

| 리스크 | 대응 방안 |
|--------|----------|
| **조직 내 정보 보안 우려** | 엔터프라이즈 플랜에 "관리자 승인 기능" 추가 |
| **무료 사용자만 늘고 유료 전환 저조** | 핵심 기능(e.g., 고급 템플릿)은 유료 전용 |
| **세일즈 주도형 조직과 충돌** | PLG와 세일즈 병행 전략 수립 (하단 참고) |
| **K-Factor 낮게 측정될 가능성** | B2B는 주기가 길므로 3개월 단위로 평가 |

### ⚖️ **PLG vs 세일즈 주도형 하이브리드 전략**
```
소규모 팀(10명 미만): 완전 셀프서비스 (PLG)
중견 기업(50명+): 세일즈 개입 + 무료 파일럿
대기업(500명+): RFP 대응 + Enterprise 세일즈

→ 현재 귀사는 "중견" 타겟이므로 세일즈 지원 필수
```
