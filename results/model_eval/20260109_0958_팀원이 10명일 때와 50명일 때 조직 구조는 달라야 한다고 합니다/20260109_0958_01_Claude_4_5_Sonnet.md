## 3. Step-by-Step 실행 가이드 (3주 완성 로드맵)

### **[1주차] 진단 및 설계**
#### Day 1-2: 의사결정 매트릭스 작성
```
┌─────────────────┬──────────┬──────────┬──────────┐
│ 결정 유형        │ 현재     │ 이상     │ 담당자   │
├─────────────────┼──────────┼──────────┼──────────┤
│ 신규 고객 계약   │ 대표     │ 영업팀장 │ VP Sales │
│ 제품 기능 우선순위│ 대표    │ CTO+PM   │ CTO      │
│ 채용 최종 결정   │ 대표     │ 각 팀장  │ 팀장+HR  │
│ 10만원 미만 지출 │ 대표     │ 담당자   │ 각 팀    │
└─────────────────┴──────────┴──────────┴──────────┘
```
**Action**: 지난 1개월간 대표님께 올라온 결정 20건을 분류하고, 80%를 위임 가능 항목으로 재배치

#### Day 3-5: 팀장 후보 선정 및 1:1 면담
- **CTO 후보**: 기술 스택 결정 + 아키텍처 설계 경험자
- **VP Sales 후보**: 대표님 부재 시 고객 미팅 주도 가능자
- **COO 후보**: 내부 프로세스 정비 능력 보유자

> ⚠️ **리스크**: 25명 규모에서 C-level 3명은 과도할 수 있음 → 초기엔 "팀장" 직급으로 시작 후 30명 이상 시 승격 추천

---

### **[2주차] 파일럿 운영**
#### 1단계: 권한 위임 실험 (Mini-Empowerment Test)
```
[실험 설계]
- 대상: 영업팀장 후보 1명
- 기간: 2주
- 위임 범위: 
  ✓ 300만원 이하 계약 최종 승인
  ✓ 고객 불만 처리 (환불 포함)
  ✓ 팀원 일정 조율
  
- 보고 규칙:
  • 일일: Slack 요약 (5분)
  • 주간: 1:1 미팅 (30분)
  • 긴급: 즉시 전화
```

#### 2단계: 프로세스 문서화
- **결재 플로우 차트** 제작 (Notion/Confluence)
- **Exception Rule** 명시: "이럴 땐 대표에게 보고"
  - 예: 1,000만원 이상 계약 / 법적 리스크 / 언론 대응

---

### **[3주차] 전사 확대 적용**
#### 조직 개편 공지 및 Kick-off
```
[공지 예시 구조]
1. Why: "더 빠른 실행을 위해"
2. What: 새로운 팀 구조 발표
3. How: 각 팀의 결정 권한 범위
4. Safety Net: "실수는 대표가 책임, 도전은 팀이 주도"
```

#### 주간 리더십 미팅 정례화
- **시간**: 매주 월요일 오전 10시 (1시간)
- **참석**: 대표 + 3팀장
- **안건**: 
  1. 각 팀 주간 목표 공유 (10분씩)
  2. 크로스 이슈 조율 (20분)
  3. 대표의 전략 방향 공유 (10분)

## 6. 장기 로드맵 (6개월 View)

```

  → 3팀장 체제 정착
  → 대표의 의사결정 건수 70% 감소 확인
  
[Month 3-4] 프로세스 고도화
  → 권한 위임 매뉴얼 2.0 버전 제작
  → 팀장 평가 지표 도입 (팀 성과 + 리더십)
  
[Month 5-6] 스쿼드 전환 준비
  → 50명 도달 시 Product Squad 본격 분리
  → CTO 산하 "프랜차이즈 전담팀" vs "개인식당 전담팀" 분화
```

## 7. 후속 질문 제안 🔄

대표님의 조직 성장을 위해 다음 주제도 함께 고민하시면 좋습니다:

1. **"팀장 후보들에게 어떤 교육이 필요할까요?"**
