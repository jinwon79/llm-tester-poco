## 🎯 3. Step-by-Step 실행 가이드

### **Phase 1: 현재 시스템 최적화 (지금~3개월)**

| 항목 | 실행 내용 | 기대 효과 |
|------|-----------|----------|
| **프롬프트 엔지니어링** | GPT-4 사용량 30% 절감 가능한 프롬프트 설계 | 월 비용 즉시 감소 |
| **캐싱 시스템 구축** | 유사 질문은 DB 저장 후 재사용 | API 호출 40% 감소 |
| **데이터 수집 파이프라인** | 사용자 피드백(좋아요/수정 요청) 자동 저장 | 파인튜닝용 데이터셋 확보 |
| **데이터 보안 강화** | OpenAI Zero Data Retention 정책 적용 + 개인정보 마스킹 | 데이터 유출 리스크 70% 감소 |

**즉시 실행 가능한 액션**:
1. OpenAI API 설정에서 `data retention opt-out` 활성화 (무료)
2. Redis 또는 PostgreSQL 기반 응답 캐시 시스템 구축 (개발 2주)
3. 학생별 오답 패턴 + 생성 문제 품질 평가 데이터 수집 시작

---

### **Phase 2: 하이브리드 모델 테스트 (3~9개월)**

#### 📌 오픈소스 모델 후보 분석

| 모델 | 장점 | 단점 | 권장 시나리오 |
|------|------|------|--------------|
| **Llama 3 (8B)** | 상업적 이용 가능, 한국어 파인튜닝 사례 다수 | 긴 문맥 처리 시 품질 저하 | 단답형 문제 생성 |
| **SOLAR (10.7B)** | 국내 개발(업스테이지), 한국어 최적화 | 라이선스 비용 발생 가능 | B2B 고객용 프리미엄 기능 |
| **Mistral (7B)** | 추론 속도 빠름, 수학 문제 강점 | 한국어 품질은 추가 파인튜닝 필수 | 수학/과학 영역 특화 |

**실행 단계**:
1. **PoC(Proof of Concept)**: Llama 3 8B 모델로 100개 샘플 문제 생성 테스트
   - 예상 비용: GPU 서버(A100) 클라우드 임대 월 50만원
   - 기간: 1개월
2. **품질 비교**: GPT-4 vs 파인튜닝 모델의 문제 품질을 학원 테스터 10명에게 블라인드 평가
3. **의사결정 기준**: 파인튜닝 모델이 GPT-4 대비 80% 이상 품질 도달 시 전환 고려

---

### **Phase 3: 자체 모델 전환 (9~18개월)**

#### 🚀 완전 전환 조건 체크리스트
- [ ] 파인튜닝 모델이 GPT-4 품질의 85% 이상 달성
- [ ] 월 사용자 5천 명 이상 확보 (규모의 경제 달성)
- [ ] AI 인프라 운영 엔지니어 1명 추가 채용 완료
- [ ] 데이터 품질 관리 프로세스 정립

#### 💰 비용 시뮬레이션 (월 기준)

| 구분 | OpenAI API (현재) | 자체 모델 (목표) | 절감액 |
|------|------------------|----------------|--------|
| **소규모 (1천 사용자)** | 약 100만원 | 약 120만원* | ❌ 불리 |
| **중규모 (5천 사용자)** | 약 500만원 | 약 180만원 | ✅ 320만원 |
| **대규모 (2만 사용자)** | 약 2,000만원 | 약 300만원 | ✅ 1,700만원 |

*초기 GPU 서버 + 유지보수 인력 포함

**💡 인사이트**: 사용자 5천 명 이하에서는 API가 유리, 그 이상부터 자체 모델이 압도적 우위
