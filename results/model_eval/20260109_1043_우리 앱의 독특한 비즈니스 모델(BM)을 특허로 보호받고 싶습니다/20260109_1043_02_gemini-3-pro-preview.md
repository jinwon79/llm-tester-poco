### [상황 진단]
*   **기업명:** (주)핀포인트
*   **핵심 모델:** 유휴 주차 공간 실시간 경매 시스템 (Auction-based Parking Sharing)
*   **진단:**
    *   대표님이 기획자 출신이므로 '비즈니스 로직(경매 규칙)'을 특허의 핵심으로 생각하기 쉽습니다.
    *   하지만 특허청 심사 기준상, **단순한 경제 법칙이나 인위적인 약속(룰)은 특허 대상이 아닙니다.**
    *   **결론:** 기술적 요소(하드웨어, 서버 프로세스, 통신 기술 등)가 반드시 결합되어야만 등록 가능합니다.

### [핵심 솔루션: BM 특허 등록을 위한 3대 요건]

질문하신 내용에 단호히 답변드리자면, **"네, 기술적 요소가 반드시, 그리고 구체적으로 포함되어야 합니다."**

BM 특허(Business Model Patent)는 **[비즈니스 아이디어 + 정보통신기술(ICT)]**의 결합입니다. 핀포인트가 특허를 등록받기 위해 충족해야 할 구조는 다음과 같습니다.

| 구분 | 특허가 안 되는 것 (순수 아이디어) | 특허가 되는 것 (기술적 사상) |
| :--- | :--- | :--- |
| **개념** | "주차장을 경매로 가장 비싸게 부른 사람에게 빌려주자." | "위치 기반으로 500m 내 사용자 단말에 경매 개시 신호를 송신하고, 3분 내 수신된 입찰 데이터를 서버가 정렬하여 낙찰자를 선정하는 처리 과정" |
| **핵심** | 사람 간의 약속, 경제 원칙 | **시계열적 데이터 처리, 하드웨어(서버, DB, 단말기)의 동작** |

**핀포인트 맞춤 전략:**
'경매'라는 행위 자체는 공지된 사실입니다. 따라서 특허의 포인트는 **'주차 공간이라는 특수한 시공간적 제약 상황에서 경매가 이루어지는 기술적 메커니즘'**에 맞춰야 합니다.

### [Step-by-Step 실행 가이드: 특허청을 설득하는 논리 구축]

이미 파트너 변리사가 계시므로, 기획팀과 개발팀은 다음 자료를 준비하여 변리사에게 전달해야 등록 확률이 비약적으로 높아집니다.

#### 1단계: '구성요소'의 기술적 구체화 (Planning to Tech)
기획서의 용어를 기술 용어로 변환하십시오.
*   **[기획]** 건물주가 남는 자리를 올린다. $\rightarrow$ **[특허]** 제1 단말(건물주)이 주차 가능 시간 및 최저 입찰가 데이터를 생성하여 메인 서버로 전송한다.
*   **[기획]** 운전자들이 입찰한다. $\rightarrow$ **[특허]** 서버는 GPS 기반 반경 N km 내의 제2 단말(운전자)들을 필터링하여 경매 UI를 팝업하고, 입력된 입찰 패킷을 수신하여 DB에 실시간 갱신한다.

#### 2단계: '진보성' 확보를 위한 차별점 명시
단순 경매는 거절될 확률이 높습니다. 핀포인트만의 로직을 기술과 엮으십시오.
*   **예시:** 입찰 마감 5분 전 트래픽 폭주 시 서버 부하 분산 처리 기술, 혹은 노쇼(No-Show) 방지를 위한 위치 인증 기반 낙찰 확정 로직 등.

#### 3단계: 권리 범위 설정 (Claim Strategy)
*   개발자 2명과 협의하여, 실제 구현될 시스템 아키텍처(순서도)를 그리십시오.
*   이 순서도가 특허 명세서의 '청구항'이 됩니다. 너무 구체적이면 회피 설계가 쉽고, 너무 추상적이면 거절됩니다. 적절한 범위를 파트너 변리사와 조율하십시오.

### [리스크 체크리스트]

1.  **추상적 아이디어 거절 위험:** "서버를 이용하여 주차장을 중개한다" 수준의 명세서는 100% 거절됩니다. '어떤 데이터'가 '어떤 순서'로 처리되는지 명시해야 합니다.
2.  **선행기술 존재 여부:** '주차', '공유', '예약' 관련 특허는 이미 많습니다. '실시간 경매'라는 키워드가 결합된 선행 특허가 있는지 윕스(WIPS)나 키프리스(KIPRIS)에서 반드시 검색해야 합니다. (파트너 변리사에게 정밀 검색 요청 필수)
3.  **해외 출원 시기:** 앱 출시 전(공지 전)에 국내 출원을 완료해야 하며, 해외 진출 계획이 있다면 국내 출원일로부터 1년 이내에 PCT 출원 등을 진행해야 우선권을 인정받습니다.
