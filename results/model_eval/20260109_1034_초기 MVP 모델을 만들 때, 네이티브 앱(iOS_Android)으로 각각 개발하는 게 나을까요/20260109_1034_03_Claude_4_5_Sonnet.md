## ✅ [핵심 솔루션: 크로스 플랫폼 권장]

### **결론: Flutter 기반 크로스 플랫폼 개발 추천**

| 기준 | 네이티브 (Swift/Kotlin) | 크로스 플랫폼 (Flutter) | 크로스 플랫폼 (RN) |
|------|------------------------|------------------------|-------------------|
| **개발 속도** | ❌ 느림 (각각 개발) | ✅ 빠름 (1회 개발) | ⚠️ 보통 |
| **인력 효율** | ❌ 2개 팀 필요 | ✅ 1명으로 가능 | ⚠️ 1명 가능하나 버그 많음 |
| **카메라/AI 성능** | ✅ 최상 | ⚠️ 양호 (플러그인) | ❌ 불안정 |
| **유지보수** | ❌ 이중 관리 | ✅ 단일 코드베이스 | ⚠️ 버전 의존성 높음 |
| **초기 비용** | ❌ 높음 | ✅ 낮음 | ✅ 낮음 |
| **MVP→양산 전환** | ⚠️ 처음부터 재개발 | ✅ 점진적 개선 가능 | ❌ 성능 이슈 시 재개발 |

### 🎯 홈핏에 Flutter를 추천하는 3가지 이유

1. **시간 압박 해소**: 3개월 내 iOS/Android 동시 출시 가능
2. **1인 개발 최적화**: 단일 코드로 양쪽 플랫폼 관리
3. **AI 기능 구현 가능**: TensorFlow Lite 플러그인으로 자세 인식 구현 검증됨 (예: Nike Training Club 유사 사례)

## 🛠️ [Step-by-Step 실행 가이드]

### Phase 1: 기술 검증 (1주)
**Day 1-3**: Flutter 환경 구축 + 카메라 플러그인 테스트
```bash
# 필수 플러그인
- camera: ^0.10.0  # 실시간 카메라 접근
- tflite: ^1.1.2   # AI 모델 로딩
- provider: ^6.0.0 # 상태 관리
```

**Day 4-7**: AI 자세 교정 PoC (Proof of Concept)
- Google의 MoveNet 모델 활용 (오픈소스)
- 스쿼트 1개 동작만 우선 구현하여 기술 타당성 검증

### Phase 2: MVP 개발 (8주)
| 주차 | 작업 내용 | 체크포인트 |
|------|-----------|-----------|
| 1-2주 | UI/UX 기본 틀 | 온보딩 + 운동 선택 화면 |
| 3-5주 | 카메라 + AI 통합 | 실시간 자세 피드백 |
| 6-7주 | 백엔드 연동 | 운동 기록 저장 (Firebase 추천) |
| 8주 | 테스트 + 배포 | TestFlight/Play 베타 |

### Phase 3: 시장 검증 (3주)
- **타겟 100명** 베타 테스터 모집 (인스타그램 광고 30만원)
- **핵심 지표**:
  - DAU (일 활성 사용자)
  - 운동 완료율
  - 자세 교정 만족도

- 단, AI 모델 통합 시 성능 이슈 사전 검증 필수
- 참고 사례: 'Freeletics' 앱 (RN으로 구현)

### 투자 유치 후 네이티브 전환 계획
```
MVP (Flutter) → PMF 검증 (6개월) 
→ 투자 유치 (시드) → 네이티브 재개발 (iOS 우선)
```
- 네이버, 당근마켓도 초기엔 하이브리드 → 네이티브 전환

## 📌 [후속 질문 제안]

홈핏의 성공적인 런칭을 위해 추가로 고민해보시면 좋을 주제들입니다:
